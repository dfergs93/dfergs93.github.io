Commands
/p /protocol
/p TAVR
Opens the matching CT/MRI protocol page
/g /guide
/g fleischner 6mm solid low risk
Opens matching guideline — pre-fills calculator fields if query includes numbers
/r /resource
/r cardiac sarcoidosis
Opens local resource page; falls back to Radiopaedia article via AI if no local match
/rs /rpsearch
/rs cardiac amyloid
Opens Radiopaedia search results directly — bypasses local resources
/a /anatomy
/a coronary artery
Opens IMAIOS e-Anatomy search
/c /context
/c what are the key findings?
Reads the current page and answers in the overlay — no navigation
/cmp /compare
/cmp TAVR and TMVR
Opens side-by-side protocol comparison — also: vs, versus, &, comma
/f /feedback
/f the TAVR link was wrong
Sends feedback to the developer
Auto-routing
No command? Just type naturally — what is the cardiac CT protocol for TAVI? — and Claude classifies your intent and picks the right track automatically. This is a fallback, not a command.
AI-generated answers
Add ? to any navigation query — /p TAVR?, /g fleischner 6mm?, /r cardiac amyloid? — and ORCA opens the target page and generates a Claude-powered answer from its content in the overlay. The source page is always open so you can read the full context or verify the answer directly. No extra steps — one query does both.
Radiopaedia 404 correction workflow
Step 1
ORCA detects a 404
The AI-guessed article slug didn't match a real page. ORCA opens the Radiopaedia search results for your query instead.
→
Step 2
Find the correct article
Navigate to the correct Radiopaedia article. The overlay will prompt you to paste the URL.
→
Step 3
Paste the URL
Paste the correct article URL into the overlay and press Enter. ORCA navigates there immediately.
→
Step 4
Saved and synced
The correction is cached locally and submitted to the team. Future queries go direct — no 404, no prompt.
Corrections sync across the pilot group once approved. Use /rs <query> to search Radiopaedia directly at any time without triggering the correction flow.
