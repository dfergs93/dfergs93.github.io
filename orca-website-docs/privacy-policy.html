ORCA Privacy Policy

Effective date: April 10, 2026
Product: ORCA — Orchestrated Radiology Clinical Assistant (Chrome Extension)

This policy describes how the ORCA Chrome extension collects, uses, stores, and shares information when you use it.

1. What Data We Collect

ORCA collects the following data during normal use:

Data TypeWhen CollectedPurpose
Pilot access token When you enter it in the Settings page Authenticates your access to the ORCA backend service
Query text Each time you type a command or question into the overlay Sent to the ORCA server to route your request and generate an answer
Current page text (up to 4,000 characters) Only when your query contains a question mark (?) Provides context so the server can generate a page-aware answer
Feedback text Only when you explicitly submit feedback via the feedback button Used to improve ORCA's routing and answers
No patient data: ORCA does not collect patient names, medical record numbers, dates of birth, diagnostic images, or any other protected health information. Do not use ORCA's context-capture feature (queries with ?) while viewing pages that display patient records.

2. How We Use Your Data

  • Pilot token: Verified server-side on each request to confirm you are an authorized pilot user. Not stored in server logs.
  • Query text: Processed in real time to classify your intent, match against radiology resources, and optionally generate an AI-powered answer via the Anthropic Claude API. Query text may appear in server logs for debugging purposes.
  • Page text context: Sent as part of the request payload when you ask a question with ?. It is used only to generate the answer for that specific request and is not stored beyond the request lifecycle.
  • Feedback text: Stored in persistent server-side storage to help improve routing accuracy and answer quality.

3. Data Storage

  • Pilot token (local): Stored in Chrome's built-in sync storage (chrome.storage.sync), which is encrypted by Chrome and may sync across your signed-in Chrome devices. ORCA does not have access to this storage outside of the extension context.
  • Server-side data: The ORCA backend runs on Fly.io infrastructure. Query logs are retained temporarily for debugging. Feedback submissions are stored in persistent server-side storage.
  • Page context: Not persisted. Used only within the request that triggered it and discarded immediately after the response is returned.

4. Data Sharing

ORCA does not sell your data. Data is shared only with the following third parties as necessary to operate the service:

Third PartyData SharedPurpose
Anthropic (Claude API) Query text; page context (when applicable) AI-powered answer generation for natural-language queries
Fly.io All server-bound data (see above) Cloud infrastructure and database hosting
National Library of Medicine (PubMed API) Search terms derived from your query Literature search via the /lit command

No data is shared with advertisers, data brokers, or other third parties.

5. Data Retention

  • Pilot token: Retained in Chrome sync storage until you clear it via the Settings page or uninstall the extension.
  • Query logs: Retained on Fly.io servers for a limited period for debugging, then purged automatically.
  • Feedback submissions: Retained on the ORCA server in persistent storage to support ongoing improvement.

6. Your Rights and Choices

  • Delete your token: You can clear your pilot token at any time from the ORCA Settings page (Cmd+Shift+Space → Settings icon), or by uninstalling the extension.
  • Feedback deletion: Contact us at the address below to request deletion of any feedback you have submitted.

7. Security

Pilot tokens are stored using Chrome's encrypted sync storage. All communication between the extension and the ORCA backend uses HTTPS. API keys and credentials are stored as server-side environment secrets (not in extension code or the repository).

8. Changes to This Policy

If this policy changes materially, the effective date above will be updated. Continued use of the extension after a policy update constitutes acceptance of the revised policy.
